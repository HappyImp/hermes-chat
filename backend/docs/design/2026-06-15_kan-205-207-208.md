# 设计文档：KAN-205/207/208 实现

## 架构概览

```
请求 → auth_middleware → TenantScope 提取 → handler → service → DB
                                   ↑
                          X-Tenant-ID / ?tenant= / default
```

## 变更清单

### 1. 数据库层（db/pool.rs）
- `run_migrations` 新增：`ALTER TABLE permissions ADD COLUMN tenant TEXT NOT NULL DEFAULT 'default'`
- 使用 `let _ =` 忽略已存在列的错误（兼容旧数据库）

### 2. 模型层（models/permission.rs）
- `Permission` 新增 `tenant: String` 字段
- `SetPermission` 新增 `tenant: String` 字段
- `PermissionResponse` 新增 `tenant: String` 字段

### 3. 中间件层（middleware/tenant.rs）— KAN-208
- `TenantScope` 结构体：持有 `tenant_id: String`
- 实现 `FromRequestParts` trait，自动从请求中提取 tenant
- 提取优先级：header > query param > default
- `is_valid_tenant_id()` 验证格式
- 4 个单元测试

### 4. 服务层
- `ProfileService`（services/profile.rs）— KAN-205
  - `list_profiles()` — 读取 profiles 目录
  - `list_profiles_json()` — 返回 JSON 格式
  - `read_profile_description()` — 从 SKILL.md/README.md 推导描述
  - 5 个单元测试

- `EmployeeService`（services/employee.rs）— KAN-207
  - `list_allowed()` — 保持向后兼容（默认 tenant）
  - `list_allowed_for_tenant()` — 按 tenant 过滤
  - `list_all_tenants()` — 返回所有 tenant 的权限

- `AdminService`（services/admin.rs）— KAN-207
  - `update_user_permissions()` — 向后兼容，默认 tenant
  - `update_user_permissions_for_tenant()` — 指定 tenant
  - `get_user_detail()` — 权限返回带 tenant 信息

- `AuthService`（services/auth.rs）— KAN-207
  - `register()` — 权限写入时带 tenant='default'
  - 同步写入 user_tenants 映射

### 5. 处理器层（handlers/employee.rs）
- `list()` — 新增 TenantScope 提取器 + all_tenants 参数
- `list_profiles()` — 新增 handler

### 6. 路由（main.rs）
- `GET /api/employees/profiles` — 新增路由
- AppState 新增 `profile_service: ProfileService`

## 向后兼容性
- `permissions` 表新增列有默认值，现有数据自动迁移
- `list_allowed()` 保持原接口不变
- `update_user_permissions()` 保持原接口不变
- 管理员 API 的 `UpdatePermissionsRequest` tenant 字段有默认值

## 测试策略
- 单元测试：tenant 验证逻辑、YAML 解析、markdown 解析
- 集成测试：需要真实数据库，由后续任务执行
