# F2 会话管理 + F3 Channel 管理 + F5 导出 PRD

**日期**: 2026-06-14
**迭代**: v1.1 — v1.2
**状态**: ✅ 已完成

## 1. 功能描述

### F2: 会话管理
支持创建、切换、删除会话，每个会话独立维护消息历史。

### F3: Channel 管理
支持多个 Channel（频道），每个 Channel 独立管理一组会话，实现多项目/多场景隔离。

### F5: 导出对话
将当前会话导出为 Markdown 文件下载。

## 2. 用户故事

### F2 会话管理
- 作为用户，我希望在侧边栏看到当前 Channel 的会话列表，快速切换上下文
- 作为用户，我希望新建会话来开始一个全新的话题
- 作为用户，我希望删除不需要的会话以保持列表整洁
- 作为用户，我希望会话按更新时间倒序排列，最近的对话在最上面
- 作为用户，我希望会话标题自动取自第一条消息（截取前 20 字符），这样一眼就知道每个会话在聊什么

### F3 Channel 管理
- 作为用户，我希望创建多个 Channel 来区分不同项目或场景
- 作为用户，我希望删除 Channel 时其下所有会话也一并清除
- 作为用户，我希望默认有个 "default" Channel，不需要手动创建

### F5 导出对话
- 作为用户，我希望点击导出按钮下载当前会话的 Markdown 文件，方便归档或分享
- 作为用户，我希望导出文件包含完整的对话记录和导出时间

## 4. 技术约束

- **状态模型**: `sessions: Record<string, Session[]>`（channel → sessions 映射）
- **数据模型**: Session = `{id, title, channel, messages, createdAt, updatedAt}`
- **导出格式**: Markdown，文件名 `hermes-chat-{channel}-{date}.md`
- **存储**: 复用 Zustand persist，与 F4 共享存储层

## 5. 优先级

| 功能 | 优先级 | 原因 |
|------|--------|------|
| F2 会话管理 | **P0** | 多会话是聊天应用的基础能力 |
| F3 Channel 管理 | **P1** | 多项目隔离提升体验但非阻塞 |
| F5 导出对话 | **P2** | 锦上添花，优先级最低 |
