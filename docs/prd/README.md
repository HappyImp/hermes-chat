# Hermes Chat 产品需求文档 (PRD)

## 1. 产品概述

Hermes Chat 是一个 Web 端 AI 聊天客户端，让用户通过浏览器与 Hermes Agent 进行实时对话。

## 2. 目标用户

- 需要与 AI 助手对话的开发者和技术人员
- 需要管理多个对话上下文的用户

## 3. 功能需求

### 3.1 核心功能

#### F1: 实时聊天
- **描述**: 用户发送消息，AI 通过 SSE 流式回复
- **优先级**: P0
- **验收标准**:
  - 支持 Enter 发送，Shift+Enter 换行
  - AI 回复逐字流式展示
  - 支持 Markdown 渲染（标题、列表、代码块、表格、链接）
  - 代码块带语言标签和复制按钮

#### F2: 会话管理
- **描述**: 支持创建、切换、删除会话
- **优先级**: P0
- **验收标准**:
  - 侧边栏显示当前 Channel 的会话列表
  - 支持新建会话
  - 支持切换到历史会话
  - 支持删除会话
  - 会话按更新时间倒序排列
  - 会话标题自动取自第一条用户消息（截取前 20 字符）

#### F3: Channel 管理
- **描述**: 支持多个 Channel，每个 Channel 独立管理会话
- **优先级**: P1
- **验收标准**:
  - 默认 Channel 为 "default"
  - 支持创建新 Channel
  - 支持删除 Channel（删除其下所有会话）
  - 切换 Channel 时显示对应会话列表

#### F4: 数据持久化
- **描述**: 会话历史保存到浏览器 localStorage
- **优先级**: P0
- **验收标准**:
  - 刷新页面后会话数据不丢失
  - 使用 Zustand persist 中间件实现

#### F5: 导出对话
- **描述**: 将当前会话导出为 Markdown 文件
- **优先级**: P2
- **验收标准**:
  - 点击导出按钮下载 .md 文件
  - 文件包含完整的对话记录和导出时间

### 3.2 非功能需求

| 需求 | 描述 |
|------|------|
| 性能 | 首屏加载 < 2s，消息渲染无卡顿 |
| 兼容性 | 支持 Chrome、Firefox、Safari 最新版本 |
| 响应式 | 支持移动端（≥320px）和桌面端 |
| 无障碍 | 基本 ARIA 标签支持 |

## 4. 技术约束

- 前端框架：React 18 + TypeScript
- 构建工具：Vite
- 样式方案：TailwindCSS
- 状态管理：Zustand
- 存储：localStorage
- API：SSE 流式接口 `/chat/api/v1/chat/completions`
