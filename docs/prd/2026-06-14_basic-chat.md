# F1 实时聊天 + F4 数据持久化 PRD

**日期**: 2026-06-14
**迭代**: v1.0
**状态**: ✅ 已完成

## 1. 功能描述

### F1: 实时聊天
用户通过浏览器发送消息，AI 通过 SSE (Server-Sent Events) 流式回复，实现低延迟的实时对话体验。

### F4: 数据持久化
会话历史保存到浏览器 localStorage，刷新页面后数据不丢失。

## 2. 用户故事

### F1 实时聊天
- 作为用户，我希望输入消息后按 Enter 发送，AI 逐字流式回复，这样对话体验自然流畅
- 作为用户，我希望 AI 回复支持 Markdown 渲染（标题、列表、代码块、表格、链接），这样阅读技术内容更方便
- 作为用户，我希望代码块带语言标签和复制按钮，这样可以直接复制代码使用

### F4 数据持久化
- 作为用户，我希望关闭浏览器后再打开，之前的对话记录还在，这样不需要重复提问
- 作为用户，我希望数据持久化对性能没有明显影响

## 3. 验收标准

### F1
- [x] 支持 Enter 发送，Shift+Enter 换行
- [x] AI 回复逐字流式展示（SSE）
- [x] 支持 Markdown 渲染：标题、列表、代码块、表格、链接
- [x] 代码块带语言标签和一键复制按钮

### F4
- [x] 刷新页面后会话数据不丢失
- [x] 使用 Zustand persist 中间件实现
- [x] localStorage 不可用时降级为空状态（不报错）

## 4. 技术约束

- **API**: SSE 流式接口 `POST /chat/api/v1/chat/completions`
- **状态管理**: Zustand + persist 中间件
- **存储 Key**: `hermes_chat_sessions`（localStorage）
- **Markdown**: marked + highlight.js
- **消息格式**: OpenAI 兼容格式 `{role, content}`

## 5. 优先级

| 功能 | 优先级 | 原因 |
|------|--------|------|
| F1 实时聊天 | **P0** | 核心功能，没有聊天就没有产品 |
| F4 数据持久化 | **P0** | 刷新丢数据是不可接受的 |
