# F6: 员工异步任务 PRD

## 概述

在聊天界面中支持通过命令触发员工异步任务，并实时显示任务状态和结果。

## 功能列表

### F6.1 命令解析
- 支持 `/dispatch <员工名> <任务描述>` 格式
- 验证员工名是否有效
- 验证任务描述是否为空

### F6.2 任务调度
- 调用后端 API 启动员工任务
- 返回任务信息（ID、状态、启动时间）
- 支持任务状态轮询

### F6.3 任务卡片
- 显示任务状态（等待/执行中/完成/失败/超时）
- 显示员工头像和角色
- 显示任务描述和结果

### F6.4 状态轮询
- 每 5 秒轮询一次任务状态
- 最多轮询 5 分钟（60 次）
- 超时自动标记为失败

## 用户故事

### US1: 触发员工任务
**作为** 用户
**我想要** 在聊天中输入命令触发员工任务
**以便** 让员工异步执行复杂任务

**验收标准：**
- 输入 `/dispatch 404 修复bug` 能启动任务
- 输入无效员工名显示错误
- 输入空任务描述显示错误

### US2: 查看任务状态
**作为** 用户
**我想要** 看到任务的执行状态
**以便** 了解任务进展

**验收标准：**
- 任务启动后显示执行中状态
- 任务完成后显示完成状态
- 任务失败时显示错误信息

### US3: 查看任务结果
**作为** 用户
**我想要** 看到任务的执行结果
**以便** 了解任务产出

**验收标准：**
- 任务完成后显示结果内容
- 结果支持 Markdown 渲染
