# 员工状态面板功能 PRD

**日期**: 2026-06-14
**状态**: ✅ 已完成

## 1. 功能描述

在侧边栏底部增加员工状态面板，实时显示团队成员（老财、铁壳、小K、404、裁判君）的在线状态、当前任务和工作时长。数据来源于 `/tmp/employees-active.json`，由 Shell Hooks 自动写入。

## 2. 用户故事

- 作为项目负责人，我希望一眼看到哪些员工正在工作、在干什么，这样不需要逐个询问
- 作为用户，我希望看到员工的任务描述和已工作时长，方便判断任务进度
- 作为用户，我希望离线员工显示为灰色，在线员工显示为绿色，直观区分状态
- 作为用户，我希望状态面板自动刷新（30 秒轮询），不需要手动刷新页面

## 3. 验收标准

- [x] 侧边栏底部显示员工状态列表
- [x] 在线员工：绿色圆点 + 员工名 + 任务描述 + 工作时长
- [x] 离线员工：灰色圆点 + 员工名 + "离线" 文字
- [x] 30 秒自动轮询刷新状态
- [x] API 不可用时优雅降级（显示全部离线）
- [x] 任务描述超过 30 字符截断 + 省略号

## 4. 技术约束

- **数据源**: `GET /employees/active` → `/tmp/employees-active.json`
- **轮询间隔**: 30 秒（useEmployeeStatus hook）
- **员工列表**: 硬编码 `["老财", "铁壳", "小K", "404", "裁判君"]`
- **TTL**: 活跃记录 2 小时过期（由 hook 脚本清理）
- **降级策略**: API 失败时返回空 map，所有员工显示离线

## 5. 优先级

| 功能 | 优先级 | 原因 |
|------|--------|------|
| 员工状态面板 | **P1** | 提升团队协作可视化，非阻塞核心功能 |
| 自动刷新 | **P1** | 手动刷新体验太差 |
| 离线降级 | **P1** | 保证即使后端异常也不影响页面使用 |
