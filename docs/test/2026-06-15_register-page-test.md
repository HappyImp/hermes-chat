## Bug 列表

### Bug #1: [严重] 注册API字段名不匹配

**复现步骤**:
1. 打开注册页面
2. 切换到注册模式
3. 填写用户名: testuser
4. 填写密码: Test123456
5. 填写授权码: test-code
6. 点击注册按钮

**预期结果**: 注册成功，返回201状态码

**实际结果**: 
- 前端发送: `{"username":"testuser","password":"Test123456","invitationCode":"test-code"}`
- 后端期望: `{"username":"testuser","password":"Test123456","invitation_code":"test-code"}`
- 错误信息: `Failed to deserialize the JSON body into the target type: missing field 'invitation_code'`
- HTTP状态码: 422

**影响**: 用户无法完成注册，注册功能完全不可用

**修复建议**: 
- 方案1: 修改前端代码，将`invitationCode`改为`invitation_code`
- 方案2: 修改后端代码，支持`invitationCode`字段名

### Bug #2: [一般] 密码提示不显示

**复现步骤**:
1. 打开注册页面
2. 切换到注册模式
3. 在密码输入框输入3位字符（如"abc"）
4. 观察页面提示

**预期结果**: 显示「至少 6 位」提示信息

**实际结果**: 没有显示任何提示信息

**影响**: 用户不知道密码长度要求，可能导致注册失败

**修复建议**: 
- 检查前端密码验证逻辑
- 确保输入3位密码时显示「至少 6 位」提示
