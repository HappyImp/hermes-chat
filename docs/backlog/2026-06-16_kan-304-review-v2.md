## 🟡 中等问题（建议修复）

### 1. `task_claimed` 处理未重算旧 assignee 状态

**文件**: `src/hooks/useEmployeeStatus.ts:337-348`
**问题**: `handleWsEvent` 中 `task_claimed` 分支不记录 `oldAssignee`。当任务从 A 重新分配到 B 时，后端发送 `task_claimed`（status 不变但 assignee 变更），前端只重算 B 的状态，不重算 A 的状态。A 的面板会残留已转移的任务数据。

`task_changed` 分支正确记录了 `oldAssignee` 并双端重算（line 346），但 `task_claimed` 分支遗漏了。

**修复方案**:
```typescript
} else if (type === 'task_claimed') {
  // 记录旧 assignee 用于双端重算
  if (idx !== -1) {
    oldAssignee = mapKanbanAssigneeToEmployee(tasks[idx].assignee);
  }
  kanbanTasksRef.current = idx === -1
    ? [...tasks, task]
    : [...tasks.slice(0, idx), task, ...tasks.slice(idx + 1)];
}
```

**文件**: `src/hooks/useEmployeeStatus.ts:264-303` vs `141-187`
